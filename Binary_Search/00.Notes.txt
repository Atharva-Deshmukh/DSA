Pattern 3: Binary Search on Answer (min of max / max of min / feasibility check)

           When we have something like this [X, X, X, X, X, ✓, ✓, ✓, ✓, ✓]
                                                            |
                                                           ans

                                            [✓, ✓, ✓, ✓, ✓, X, X, X, X, X]
                                                           |
                                                          ans

           i.e. after a particular range, we either start getting answer, or don't get answers
           we can think of binary search, because here, our search space is from [1 --- INT_MAX] => sorted


           This also called OPPOSITE POLARITY BINARY SEARCH concept

                                         [✓, ✓, ✓, ✓, ✓, X, X, X, X, X]    --> Before BS
                                          |                            |
                                         low                          high

                                         [✓, ✓, ✓, ✓, ✓, X, X, X, X, X]    --> After BS
                                                        |  |
                                                      high low

        After BS, the low pointer (pointing to solution regions) now points to non-solution regions


        Where this is useful?

        Say, we have k = 5 but array = [1, 1, 1, 3, 6]
        k is not there in the array but we need a range where k can be there
        We just apply BS and when low > high, BS ends
        After BS ends, we are left with a range where k will definitely lie

                             0  1  2  3  4
                            [1, 1, 1, 3, 6]   mid = 2
                             |     |     |    missing[mid] < k, low = mid + 1
                             l     m     h

                             0  1  2  3  4
                            [1, 1, 1, 3, 6]   mid = 3
                                      |  |    missing[mid] < k, low = mid + 1
                                      m  h
                                      |
                                      l

                             0  1  2  3  4
                            [1, 1, 1, 3, 6]   mid = 4
                                         |    missing[mid] > k, high = mid - 1
                                         m
                                         |
                                         l
                                         |
                                         h

                             0  1  2  3  4
                            [1, 1, 1, 3, 6]  
                                      |  |    
                                      h  m
                                         |
                                         l

                    here k lies between [a[h], a[l]]

                            0  1  2  3

now when k = 2, it doesn't lie between any of the above missing[] since missing[0] === missing[n - 1] === 0
